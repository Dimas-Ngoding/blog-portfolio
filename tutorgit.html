// Tutorial Git

Git adalah pengelola source code yang sudah dikenal hampir oleh seluruh programmer. Git ini akan membuat repository / bisa diungkapkan dengan penyimpanan lokal. Di sini kalian akan belajar cara instalasi dan kegunaan dasar git. Jika kalian sama sekali tidak tahu tentang git kalian bisa tonton video ini Klik Disini

// Kenapa Git?

  • Git adalah sistem kontrol versi yang paling umum digunakan
  • Git melacak perubahan yang Anda buat pada file, sehingga Anda memiliki catatan tentang apa yang telah dilakukan, dan Anda dapat kembali ke versi tertentu jika diperlukan.
  • Git juga membuat kolaborasi lebih mudah, memungkinkan perubahan yang dilakukan oleh banyak orang untuk digabungkan menjadi satu sumber
  • Git sudah menjadi skill wajib bagi para programmer

// Memulai

Pertama-tama pastinya kalian download dulu gitnya. Jika sudah kalian bisa buat folder dimanapun, lalu klik kanan pada foldernya. Jika sudah muncul tulisan git bash here maka kalian sudah berhasil menginstall gitnya.


// Fungsi Dasar

Pertama-tama kita jadikan dulu folder ini sebagai local repository git init kalian bisa ketikan ini di git bash kalian untuk menjadikan folder kalian sebagai local repository

Langkah selanjutnya ini digunakan untuk mengcommit / menyimpan histori perubahan yang kalian lakukan. Pertama-tama kalian coba menambah file atau folder terlebih dahulu, lalu kalian save. Setelah itu pastinya di nama folder atau file kalian akan terhighlight karena perubahan yang kalian lakukan belum terdaftar di dalam git.


Nah cara untuk menambahkannya di git, kita bisa lakukan 2 langkah ini yang pertama kalian ketik git add (nama file) jika file / folder kalian sudah terlalu banyak kalian bisa langsung menambahnya sekaligus dengan cara git add . Belum selesai disitu kita harus menambahkannya ke commit git commit -m '(Pesan Kalian)' Jika kalian tidak paham mungkin istilahnya seperti ini.

Repository adalah sebuah stasiun kereta. Kita melakukan git add . bisa diibaratkan dengan penumpang yang sedang menunggu keretanya. Lalu git commit bisa diibaratkan dengan penumpang yang tadi menunggu, sudah masuk ke dalam kereta dan akan melewati stasiun-stasiun lainnya.
